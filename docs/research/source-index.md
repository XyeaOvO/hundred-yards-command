## 4. 这些资料在项目里怎么用

- 官方规则：用于建立术语表和核心机制边界。
- Basic Training：用于确定 MVP 优先级，先实现 action / reaction / fire resolution 主环。
- Playbook 与文章示例：用于设计网页端日志和交互流程。
- 样例地图与 countersheet：用于 UI 风格和战场布局参考。

## 5. 使用边界

本项目只抽象公开资料里的机制，不复制商业任务文字和完整桌游内容。  
数字版规则实现以“软件工程课程原型”所需的最小机制为目标，偏重演示和架构，不追求 100% 完整复刻。
