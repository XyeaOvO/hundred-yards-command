# 项目章程

## 1. 项目背景

本项目面向软件工程课程大作业，目标是在有限周期内完成一个具备兵棋推演核心机制的 Web 游戏原型，并配套形成规范的软件工程文档与清晰的项目组织结构。

## 2. 项目愿景

构建一个“**可学习、可演示、可扩展**”的排级战术兵棋 Web 原型：

- 可学习：降低传统兵棋入门难度
- 可演示：支持课堂汇报与现场操作
- 可扩展：为多人联网、复盘系统和地图编辑器预留空间

## 3. 项目目标

### 3.1 业务目标

- 让用户在浏览器中体验简化战术推演
- 将规则执行过程转化为可见的操作反馈与日志

### 3.2 工程目标

- 建立前端、后端、共享领域模型的清晰边界
- 输出完整的软件工程文档
- 保证原型可启动、可构建、可测试

## 4. 干系人

| 角色 | 关注点 |
| --- | --- |
| 课程教师 | 过程规范、架构清晰度、可运行程度 |

## 6. 项目范围

### In Scope

- 示例战场与单位部署
- 单位激活、移动、射击
- 地形与视线的简化判定
- 回合切换与胜负统计
- Web 前后端原型
- 软件工程文档与图表

### Out of Scope

- 付费桌游完整内容搬运
- 复杂外交/战役级系统
- 商业化发行版本
- 大规模在线联机

## 7. 成功判定标准

### 功能层

- 可以创建并运行一局示例对局
- 支持地图点击交互
- 支持日志与结果展示

### 工程层

- 目录结构清晰
- 文档和代码一致
- `npm install` 后可正常运行

### 汇报层

- 初期汇报可展示背景、目标、架构、范围与计划
- 中期汇报可展示更多机制和可测试性
- 结题汇报可展示完整功能闭环和总结

## 8. 约束与假设

### 约束

- 课程周期有限
- 项目需避免复制商业规则文本与付费内容
- 团队人力有限，优先保证主循环完整

### 假设

- 当前阶段以桌面浏览器为主要运行环境
- 原型先采用单机共享屏幕演示，不优先解决真实网络对战
- 后续如果需要扩展，可引入数据库和 WebSocket
