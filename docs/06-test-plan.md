# 测试计划

## 1. 测试目标

- 验证核心规则不会出现明显逻辑错误
- 验证接口在正常和异常输入下都能返回合理结果
- 验证前端操作流程满足课堂演示需求

## 2. 测试分层

### 单元测试

对象：`packages/shared`

重点：

- 对局初始化
- 相邻格移动
- 视线阻挡判断
- 射击结算边界
- 回合切换

### 接口测试

对象：`apps/server`

重点：

- 场景列表返回
- 对局创建
- 命令校验失败
- 非法单位操作

### 前端联调测试

对象：`apps/web`

重点：

- 初始页面加载
- 地图点击交互
- 日志展示是否同步更新
- 异常提示是否显示

## 3. 当前已落地测试

当前仓库已包含共享规则层的基础测试：

- 创建对局
- 选择并移动单位
- 树林阻断视线

## 4. 建议补充测试用例

| 编号 | 用例 | 预期结果 |
| --- | --- | --- |
| TC-01 | 选择敌方单位 | 返回规则错误 |
| TC-02 | 移动到被占据格 | 返回规则错误 |
| TC-03 | 射击超射程目标 | 返回规则错误 |
| TC-04 | 行动点耗尽后继续操作 | 返回规则错误 |
| TC-05 | 回合结束后单位重置 | 可重新激活 |
| TC-06 | 一方全灭 | 对局结束并判定胜者 |

## 5. 执行方式

```bash
npm test
```

## 6. 质量门槛建议

- 共享规则层新增逻辑必须补单元测试
- 关键 API 至少具备正常路径 + 错误路径测试
- 每次阶段演示前进行一次手工回归
