# 百码推演项目初期汇报

## 1. 项目概述

- 项目名称：百码推演 / Hundred Yards Ops
- 项目类型：软件工程课程大作业
- 产品形态：基于浏览器的兵棋推演 Web 游戏原型
- 规则来源：参考 The Last Hundred Yards 公开规则资料进行机制抽象
- 当前阶段：初期汇报 + MVP 骨架落地

## 2. 问题定义

传统兵棋桌游在教学与演示场景中存在三个明显门槛：

1. 规则复杂，初学者理解成本高。
2. 桌面对局记录难以追踪，回放与复盘不方便。
3. 课程项目时间有限，难以完整复刻商业兵棋体系。

因此本项目选择“**基于规则思想抽象的数字原型**”路线：保留兵棋推演的核心决策张力，同时降低上手门槛，突出软件工程实现与系统设计能力。

## 3. 项目目标

### 3.1 课程目标

- 形成完整的软件工程交付物，而不是单一代码仓库。
- 体现需求分析、总体设计、实现、测试与迭代规划。
- 能在课堂上进行可运行演示。

### 3.2 产品目标

- 实现一个可在网页中演示的排级战术推演原型。
- 支撑“单位激活 - 机动 - 射击 - 状态更新 - 日志记录”的主循环。
- 为后续多人对战、战报回放、任务编辑器预留扩展空间。

## 4. 当前版本范围

### 已实现

- 单一示例场景加载
- 前后端分离工程结构
- 单位选择与激活
- 相邻格机动
- 视线检查与简化射击判定
- 压制 / 消灭状态更新
- 目标区控制统计
- 回合推进与胜利判断
- 作战日志与指挥台 UI

### 暂不纳入 MVP

- 完整规则复刻
- 多人联网同步
- 账号系统
- 地图编辑器
- AI 对手
- 完整任务文本与商业棋子资源导入

## 5. 总体方案

系统采用“**React 前端 + Express API + 共享规则引擎**”三层结构：

- 前端负责地图可视化、命令交互、日志展示。
- 后端负责对局创建、命令校验、状态存储。
- 共享引擎负责核心规则计算，保证前后端语义一致。

## 7. 演示亮点

课堂初期汇报可直接演示以下内容：

1. 启动系统后自动载入示例场景。
2. 点击当前行动方单位开始激活。
3. 点击相邻地块执行机动，点击敌方单位执行射击。
4. 观察压制、消灭、行动点消耗与日志变化。
5. 展示目标区控制、回合推进和胜负判定。

## 8. 关键工程决策

### 为什么不完整复刻原桌游

- 课程周期有限，完整规则实现风险高。
- 公开资料可支持机制抽象，但不适合复制完整商业内容。
- 初期汇报更应证明“产品定义 + 架构设计 + 可运行骨架”能力。

### 为什么采用共享规则引擎

- 避免前后端各写一套规则，减少语义漂移。
- 单元测试可以直接覆盖核心规则函数。
- 后续迁移到 WebSocket 或多人模式时复用成本低。

## 9. 下一阶段计划

- 扩充规则：机会射击、领导加成、地形更多效果
- 扩充内容：更多场景、更多单位类型
- 扩充工程能力：持久化存档、回放、多人同步
- 完善测试：API 回归、前端交互测试、规则边界测试
